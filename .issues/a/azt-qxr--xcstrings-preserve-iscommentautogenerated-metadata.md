Upstream fix in Ryu0118/xcstrings-crud@716078a: `StringEntry` can carry `isCommentAutoGenerated: Bool?`. Our model does not decode it, so writes drop the flag and the next Xcode build re-evaluates the comment.

Fix in `Sources/Core/XCStrings/Models/XCStringsModels.swift`:
- Add `isCommentAutoGenerated: Bool?` to `StringEntry`
- Add to `KeyInfo` and pass through in `XCStringsReader.getKey`

Reference: https://github.com/Ryu0118/xcstrings-crud/commit/716078a

## Summary of Changes

- Added `isCommentAutoGenerated: Bool?` to `StringEntry` and `KeyInfo`.
- `XCStringsReader.getKey` passes the field through; the new `XCStringsFileEncoder` (see jig 21p-lqj) preserves it on write.
- `Tests/XCStringsUpstreamPortTests.swift` covers `getKey` passthrough and a JSON-fixture round trip via `XCStringsFileEncoder`.
